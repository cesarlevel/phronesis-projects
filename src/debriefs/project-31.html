Project 31: Orders Early Access

What made you smile?

Ability to add plans with different recurring times.

What did you find confusing?

Issues with reporting usage in the orders page and some details pages.

  • BUG New orders and active ones are not showing up in data tables.
  • SUGGESTION Add a link to the order on the timeline. At the moment, you can access the order via the tile below the customer details.
  • SUGGESTION We encounter an error when opening the order: "upcoming invoice not found." This seems normal at this point, so it should not be shown as an error to the user.

    We can see in the response to the orders call that the upcomingInvoiceId is null, so perhaps this request that results in a 404 is not needed.

    We see the same issue with a one-time order: "order not found."
  • BUG Products and plans are not showing on the recurring orders tile on the customer details page.
  • NITPICK Clicking the back button on the order details page takes you to the data tables, which should go to the previous screen.
  • NITPICK The timeline order is incorrect; subscription activated seems to come before transaction approved. This appears to be a backend issue based on the API response.
  • BUG Remove the duplicate order details. On the order details page, the edit button leads to the same place, and we already have the customer ID on the left panel. Styles for the tiles should be similar (no borders - .tile.tile-plain).
  • BUG After completing the one-time order, we see a purchase tile that is not populated with data.
  • NITPICK The usage form needs a divider between the two tile forms; the tile content text should be text-muted.
  • BUG For tiered billing, we cannot update the usage on the form; the button is not visible.
  • BUG On the order item details, we have the same issue with the repeated order details. In the correct order details tile, there are a few properties marked as n/a, which can be filtered out. Additionally, the order details have actions related to the order, not the order item. We are somewhat confused about the purpose of this view; what are the actions on the order item? We are repeating the same information that we already have on the order details page.
  • CONFUSION In the timeline, we see "subscription activated." If we click, we go to the order details page, and when we click on the order items, we seem to go to a subscription-related page, at least according to the URL. Should the timeline say "Order activated," or should we have two instances of "subscription activated" (for two items in an order)?
  • POSSIBLE BUG The invoice and upcoming invoice look good, but when editing the items on the upcoming invoice, we see the original items from the first invoice. It is unclear if this is by design. We can see that we use the order GET to fill the edit items on the upcoming invoice, but they do not match if the order has a one-time item in the initial invoice.
